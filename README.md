# Carousel Universal System

Универсальный skill для генерации Instagram-каруселей почти на любую тему.

Это не архив для разового анализа и не узкий prompt под одну нишу. Это **skill-first repository**, который нужно сначала **установить как навык**, а потом использовать в отдельном Project или рабочем чате.

Сильные стороны системы:
- intake перед генерацией;
- умная маршрутизация по типу задачи;
- photo-based режим;
- verification layer для цифр, сравнений и claims;
- anti-slop редактура;
- структурный финальный пакет.

## Правильный способ использования

### Шаг 1. Установите навык по ссылке на GitHub

Используйте репозиторий как skill-источник.

Практический формат запроса:

```text
Установи этот навык: https://github.com/bkkushbox/carousel-universal-system
```

Идея именно в этом: сначала модель или приложение Codex должны увидеть репозиторий как **skill**, а не как обычный ZIP для разбора.

### Шаг 2. После установки создайте отдельный Project

После установки навыка создайте отдельный Project для работы с каруселями.

Это нужно, чтобы:
- не смешивать карусельную работу с другими задачами;
- хранить там референсы, фото и файлы;
- работать внутри уже установленного навыка.

### Шаг 3. Дайте пользовательский запрос

Пример:

```text
Нужно создать карусель на тему DeFi, показать реальные сравнения прибыли, использовать мои загруженные фотографии, сначала задай уточняющие вопросы.
```

Что должно произойти правильно:
1. система не генерирует карусель сразу;
2. определяет тип задачи;
3. понимает, что нужен mixed mode;
4. включает и verification, и photo-based mode;
5. задаёт уточняющие вопросы;
6. только потом строит результат.

## Неправильный сценарий

Неправильно использовать репозиторий так, чтобы модель начинала:
- анализировать архив как skill-структуру;
- переходить в skill-creator mode;
- обсуждать, как устроен репозиторий, вместо создания карусели.

Если это произошло, значит навык не был корректно установлен как skill-first контур.

## Что внутри репозитория

- `SKILL.md` – ядро поведения навыка
- `skill.json` – metadata для skill-first регистрации
- `agents/openai.yaml` – конфигурация для OpenAI-совместимого skill-loading
- `START_HERE.md` – человеческий старт
- `references/` – шаблоны и чек-листы
- `project/` – вспомогательные файлы для Project после установки навыка
- `examples/` – тестовые сценарии

## Когда использовать этот skill

Используйте его, если нужно:
- сделать карусель почти на любую тему;
- использовать реальные данные и сравнения;
- использовать загруженные фото;
- получить не просто текст, а структурный production package;
- сделать mixed-mode карусель: verified data + photo-based layout.

## Чем он отличается от узких photo-only skill-репозиториев

Этот skill не ограничен только cover photo и не завязан на одну нишу.

Он рассчитан на:
- data-heavy темы;
- photo-heavy темы;
- mixed mode сценарии;
- универсальные темы, а не только финансы или личные фото;
- project-system use case после установки навыка.

## Если нужен запасной путь

Если какая-то платформа не умеет ставить skill напрямую по GitHub-ссылке, можно использовать репозиторий как набор файлов:
- `SKILL.md` как главный skill-инструктаж;
- `references/*` как reference layer;
- `project/*` как вспомогательные файлы для Project.

Но это именно запасной путь. Основной путь для этого репозитория: **сначала установить как skill, потом использовать в Project**.
